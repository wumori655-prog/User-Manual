# 操盘列表 · 运营功能说明(v1 现状版)

> **文档定位**:本文档面向运营人员,逐个说明【操盘列表】模块中每个按钮的功能用途、操作步骤、操作后果、权限要求与注意事项。
>
> **对应版本**:基于 GitHub master 分支(2026-05-25)`操盘列表/` 19 章完整文档
>
> **使用方法**:按位置分组导航——顶部导航 → 顶部栏 → 侧边栏 → 筛选区 → 批量操作栏 → 数据表格 → 分页 → 各类弹窗。每个按钮均含【按钮名 / 位置 / 用途 / 操作步骤 / 操作后果 / 权限 / 注意事项】7 项说明。

---

### 模块定位

操盘列表是操盘手的**主战场**,展示所有赛事的全貌。功能包括:赛事查看、筛选、上架/下架、操盘手分配、紧急锁盘、批量操作等。

**核心角色**:
- **普通操盘手**:管理自己负责的赛事
- **操盘主管**:管理本组所有赛事 + 人员分配 + 紧急操作
- **风控主管**:跨组监控 + 紧急锁盘

### 11. [🔒 一键锁盘] 紧急按钮

|---|---|

| **位置** | 顶部栏最右侧 |

| **注意事项** | 解锁原因**必填**(审计追责);与一键锁盘对称设计 |

## 📋 常见操作场景速查

### 场景 1:新赛事上架(批量)

```
步骤:
  ① 用 [待上架全部 19] 快捷按钮筛选所有待上架赛事
  ② 表头 [复选框 37] 全选 或 逐行 [复选框 38]
  ③ 底部出现批量操作栏,点 [批量上架 33]
  ④ 上架弹窗:选盘口初始状态 61(默认"跟随数据源")
  ⑤ 点 [分配操盘手 62] → 弹出分配弹窗 → 选 [按联赛默认 67] 或手动配置
  ⑥ 回到上架弹窗,点 [确认上架 63]
  
注意:操盘手必填,默认操盘手未配置时需手动选
```

### 场景 2:发现异常告警,需要紧急锁盘

```
步骤(主管/风控):
  ① 顶部栏 [一键锁盘 11] 或 快捷键 Ctrl+Shift+L
  ② 弹出异常清单(自动每 5 秒刷新)
  ③ 查看告警类型(货量大/单边超限/IM 异常等)
  ④ 勾选要锁定的赛事 [50]
  ⑤ 填写 [锁盘原因 53]
  ⑥ 点 [确认锁盘 54]

之后:
  - 处理完异常 → 用 [一键解锁 12] 恢复
  - 解锁原因必填(必须填)
```

### 场景 3:操盘手离职/请假,需要交接

```
步骤(主管):
  ① 筛选区 [操盘手 28] 下拉选该操盘手 → 查看其负责的全部赛事
  ② 表头 [复选框 37] 全选
  ③ 底部 [分配操盘手 35] → 弹出分配弹窗
  ④ 选新操盘手 → 用 [全选 70] 选所有玩法 → 点 [分配 → 72]
  ⑤ 切换到 [滚球阶段 68] Tab 重复
  ⑥ 点 [保存设置 74]
```

### 场景 4:操盘手日常工作流(开班)

```
步骤(普通操盘手):
  ① 登入后,查看顶部 [数据源状态 4] 是否正常
  ② 看 [滚球中 6] 卡片数字(有动画提示)
  ③ 点 [需处理 20] 快捷按钮,查看告警赛事
  ④ 优先处理告警 → 点行 40 进入操盘详情
  ⑤ 切回列表,点 [我负责的 21] 看自己的赛事
```

## 📌 注意事项汇总

```
🔴 高危操作(影响面广):
   - [一键锁盘 11] → 选错可能损失收益
   - [批量下架 34] → 影响客户端可见性
   - [批量分配操盘手 35] → 大量赛事人员变更

🟡 必填项:
   - 上架时 [分配操盘手 62] 必填
   - [一键解锁 12] 原因必填
   - 一键锁盘 [锁盘原因 53] 建议必填

🟢 仅查看,不可操作:
   - [总赛事 7]、[预估盈亏 8]、[服务器时间 10] 等

🔵 角色权限差异:
   - 普通操盘手:仅看自己 + 操作自己负责的赛事
   - 主管:看全组 + 人员分配
   - 风控:跨组监控 + 紧急锁盘
   
⚠️ 数据时效:
   - 列表不自动刷新,需手动操作或通过 WebSocket 推送更新
   - 异常清单弹窗每 5 秒自动刷新
   - 服务器时间 vs 本地时间偏差 > 3 秒会警告
```
