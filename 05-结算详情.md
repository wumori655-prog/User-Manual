# 结算详情 · 运营功能说明(v1 现状版)

> **文档定位**:本文档面向运营人员(结算人员/结算主管/风控人员/财务人员),逐个说明【结算详情】模块中每个按钮的功能用途、操作步骤、操作后果、权限要求与注意事项。
>
> **对应版本**:基于 GitHub master 分支(2026-05-25)`结算详情/` 11 章完整文档
>
> **使用方法**:按位置分组导航——顶部导航栏 → 左侧赛事快捷栏 → 中间赛事信息头 → 盘口数据汇总 → Tab 切换 → 盘口筛选 → 盘口结算卡片 → 右侧日志面板 → 各类弹窗。每个按钮均含【按钮名 / 位置 / 用途 / 操作步骤 / 操作后果 / 权限 / 注意事项】7 项说明。

---

### 模块定位

结算详情是结算人员的**单赛事工作台**,提供:
- **赛果录入**(按时段分组录入比分)
- **盘口结算**(逐盘口选结算结果)
- **赛事级操作**(确认结算 / 赛事驳回 / 异常处理)
- **盘口二次结算**(已结算盘口回退)

**两条结算路径**:
- **路径 A 数据源结算**:IM 自动推送终态,本地按规则自动派彩(数据源盘口默认走这条)
- **路径 B 本地结算**:可操盘盘口的赛果录入 + 人工选结果(主要工作量)

**核心角色**:
- **结算人员**:赛果录入 / 盘口结算 / 异常处理 / 二次结算 / 驳回(一线全部)
- **结算主管**:一线全部 + 人员分配(在结算列表执行)
- **风控人员**:仅查看(无操作权限,一期无导出)
- **财务人员**:仅查看

| 项 | 内容 |
|---|---|

| **位置** | 赛事信息头右侧 |

### 9. [赛事驳回] 按钮

| 项 | 内容 |
|---|---|
| **按钮名** | 赛事驳回(橙色边框) |
| **位置** | 赛事信息头(已结算/已取消/异常态时显示) |
| **用途** | 把赛事从已结算/已取消/异常态回退到待结算 |
| **操作步骤** | ① 点击 [赛事驳回] → ② 弹出驳回确认 + 原因输入 → ③ 填写原因 → ④ 确认 |
| **操作后果** | 赛事状态回到"待结算";**盘口状态不变**(需要"盘口二次结算"才能回退盘口) |
| **权限** | 结算人员 / 结算主管 |
| **注意事项** | **必须在最新结算时间起 24h 内**(UTC);超时禁用;**仅放行赛事状态,不动盘口和注单**;真正回退注单要配合 [盘口二次结算] |

### 10. [赛事异常处理] 按钮

| 项 | 内容 |
|---|---|
| **按钮名** | 赛事异常处理 |
| **位置** | 赛事信息头(仅赛事=待结算时启用) |
| **用途** | 发起 4 种异常处理:取消 / 延期 / 腰斩 / 数据错误 |
| **操作步骤** | ① 点击 [赛事异常处理] → ② 弹出异常处理弹窗 → ③ 选 4 种异常之一 → ④ 配置参数 → ⑤ 确认 |
| **操作后果** | 赛事进入异常态;数据错误不改赛事态(仅审计标记);其他 3 种会影响盘口和注单 |
| **权限** | 结算人员 / 结算主管 |
| **注意事项** | **仅赛事=待结算时启用**;异常态下此按钮**隐藏**,需先 [赛事驳回] 回待结算才能再次发起 |

```
  ⑥ 该盘口注单全部回退 + 用新结果重算 + 派彩差额处理

注意:
  - 前提:赛事=待结算;若赛事已结算,先 [赛事驳回 9]
  - 操作不可撤销,审计严格
```

### 场景 3:已结算赛事发现整体有问题需要驳回

```
步骤(结算人员,24h 内):
  ① 进入该赛事详情页(从结算列表点 [详情])
  ② 点 [赛事驳回 9]
  ③ 填写原因 → 确认
  ④ 赛事回到"待结算"
  ⑤ 此时盘口状态不变(已结算),需要 [盘口二次结算 26] 逐个回退
  ⑥ 修正错误后重新走"全场赛事正常结算"流程

注意:
  - 24h 窗口(以最新结算时间为准)
  - 赛事驳回仅放行赛事状态,不动盘口注单
```

### 场景 4:腰斩赛事处理

```
步骤(结算人员):
  ① 比赛腰斩,赛事=待结算时点 [赛事异常处理 10]
  ② 选 [异常类型 = 腰斩 45]
  ③ 选择处理方式 46:
     - 按比分结算(已完成时段结算 / 未完成退款)
     - 全额退款(所有注单退款)
     - 暂缓处理(进入异常态等待后续决策)
  ④ 点 [确认异常处理 47]
  ⑤ 所有开盘中盘口立即关盘;操盘列表联动

注意:
  - 暂缓 = 异常态(可后续驳回再处理)
  - 按比分结算 / 全额退款 = 终态(赛事变"已结算"或"已取消")
```

### 场景 5:利用 15 分钟段 6 Tab 弹窗结算

```
步骤(结算人员,该赛事有 BT48/BT179/BT180 等盘口时):
  ① 点 [📝 15 分钟段赛果录入 21]
  ② 弹出 6 Tab 弹窗
  ③ 切到 [0-15 Tab 38] → 录入该段比分 → 点该 Tab 的 [应用 39]
  ④ 切到 [15-30 Tab] → 录入 → 应用
  ⑤ ...重复 6 段
  ⑥ 关闭弹窗
  ⑦ 各段盘口可结算

注意:
  - 6 段独立,任意段都可单独录入
  - 不需要全录,可逐段录
```
